Appointments

POST /appointments HTTP/1.1

Request an appointment

Parameters
Field Type Description
user_id Integer The id of the user requesting the appointment
location_id Integer The id of the location that the user needs the appointment
service_type_id Integer The id of the service type attached to the request
in_office Boolean Will the appointment by in the office or virtual
Example
  • Request 
POST /appointments HTTP/1.1
Content-Type: application/vnd.api+json
Accept: application/vnd.api+json

{
  "data": {
    "type": "appointments",
    "attributes": {
      "user_id": 4,
      "location_id": 1,
      "service_type_id": 6,
      "in_office": 1
    }
  }
}
  • Response 
HTTP/1.1 200 OK

Create a user
POST /users HTTP/1.1

Create a user

Parameters
Field Type Description
email required String The user's email (must by valid)
firstname required String The user's first name
lastname required String The user's last name
middle_initial String The user's middle initial
ssn required String The user's Social Security Number
address_1 String The user's address (first line)
city String The user's city of residence
state String The user's state of residence
county String The user's county of residence
zip String The user's zip code
dob String The user's date of birth (Y-m-d)
Example
  • Request 
POST /users HTTP/1.1
Content-Type: application/vnd.api+json
Accept: application/vnd.api+json

{
  "data": {
    "type": "users",
    "attributes": {
      "email": "buster@bluth.co",
      "firstname": "Buster",
      "lastname": "Bluth",
      "ssn": "321214321"
    }
  }
}
  • Response 
HTTP/1.1 201 Created
